枝分かれとは

今までのプログラムの特徴:

  • プログラムが上から下へ順番に実行されていた
  • 全ての文、式が実行されていた

枝分かれの特徴:

  • 条件によってプログラムの一部だけが実行される
  • 条件によって実行されない部分もある

枝分かれの例

二つの変数 a と b の大きい方の値を変数 c に代入する

if (a >= b) {
    c = a;
}
else {
    c = b;
}

枝分かれの記述: if 文

if (/* 条件 */) {
    /* 条件が真の場合に実行する文 */
}
  • 丸括弧内に条件文を書く
  • 波括弧内に条件文が真 (正しい) の場合に実行する文を書く
  • 波括弧を省略できる場合があるが、省略しない方が安全
  • 波括弧内の文は字下げする

枝分かれの記述: if-else 文

if (/* 条件 */) {
    /* 条件が真の場合に実行する文 */
}
else {
    /* 条件が偽の場合に実行する文 */
}
  • else 後の波括弧内に条件文が偽 (間違っている) の場合に実行する文を書く
  • 波括弧の省略できる場合があるが、省略しない方が安全
  • 波括弧内の文は字下げする
  • else 後の波括弧の代わりに新たな if 文が使える (教科書 p. 96 参照)

条件の書き方: 単純な条件

参考書 8.3 節 (pp. 97-98)

  • 条件で一番多いのは二つの数字の比較
  • 比較には次の演算子を使う:
    演算子 意味
    a < b a は b より小さい、小なり
    a <= b a は b より小さいか等しい、a は b 以下
    a == b a は b と同じ、等しい
    a != b a は b と違う、等しくない
    a >= b a は b より大きいか等しい、a は b 以上
    a > b a は b より大きい、大なり

条件の組み合わせ

参考書 8.4 節 (pp. 98-102)

  • 「かつ」: 両方の条件が真でない (合わない) と全体が合わない
    a >= 20 && a <= 30
    (a は 20 以上かつ 30 以下)
    (20 <= a <= 30 とは書けない!)
  • 「又は」: どちらかの条件が真の場合全体が真
    a < 5 || a > 10
    (a は 5 より小さい又は 10 より大きい)
  • 「でない」: 真の条件を偽に、偽の条件を真にする
    !(a < 100)
    (a は 100 より小さいでない)

文字列の比較

  • 文字列の比較には関数 strcmp() を使う
  • strcmp(s1, s2) で文字列 s1 と s2 を比較する
  • 結果がゼロ以下、ゼロ、もしくはゼロ以上になる
  • 本来の比較の結果が欲しい場合は次のように書く
プログラムの記述 本来したい比較
strcmp(s1, s2) > 0 s1 > s2
strcmp(s1, s2) >= 0 s1 ≥ s2
strcmp(s1, s2) == 0 s1 == s2
strcmp(s1, s2) != 0 s1 ≠ s2
strcmp(s1, s2) <= 0 s1 ≤ s2
strcmp(s1, s2) < 0 s1 < s2

文字列の比較の意味

  • 文字列の比較は先頭から一文字づつ行われる
  • 両方の文字列の文字が同じの間は続く
  • 違う文字が見つかったら比較は決る
  • 文字の「大きさ」(順番) は文字の番号の大きさ
文字列の比較の例
s1 s2 s1 ? s2
"abc" "def" s1 < s2
"abc" "abc" s1 == s2
"abc" "abcd" s1 < s2
"abc" "abC" s1 > s2
"あおやま" "アオヤマ" s1 < s2
"アオヤマ" "青山" s1 < s2

趣旨：情報テクノロジー学科2年の段階で学んでいる講義科目を中心とした内容を、より実践的に使いこなせるように、またその理解をより深いものにするために実習を行う。机上での実習とノートPC を利用したプログラミングおよび応用プログラムを利用した実習をあわせて行う。

実習形態：テーマは原則として1 週単位、あるいは段階的に数週にわたって行われる。実習時間中に達成すべき内容とその結果をもとに検討・考察を行うべき内容を示し、実習終了時での報告とレポートとして課題の提出を行う。

実習方法：各実習においては、まず実習内容についての説明を行い、その後実習に移行する。実習の際は、グループで議論しながら進めたり、担当者に質問しながら、またそのアドバイスにしたがって進めてよい。ただし、レポートは各自で作成し提出する。

評価：
① 授業開始後15 分経過した時点で出席している学生は出席、そうでない学生は欠席とみなす。
② 合格の基準は全ての実習を行い、レポートを全て提出して合格することとし、レポートの内容により成績評価を行う。
レポートの取り扱い：当該実習のレポートは、原則として次回実習の行われる日の開始時にレポートを回収する（ただし、担当者によってその取り扱いが異なることがあるので、その指示に従うこと）。この時点で提出できない者はその次の実習開始時に提出すること。「遅刻」は1 週遅れた場合のみ認める。また、他人のレポートを写したと判断した場合には、写した学生・写させた学生両方とも採点の対象としない。
また、内容が不十分である場合には「再提出」を求めるので、各担当者の指示に従うこと。
６． レポートの書き方：レポートは、A4 サイズのレポート用紙を用い、表紙フォーマットは指示されたものを用いる。レポートの内容は、①実習内容、②課題内容とその結果、③結果についての検討および考察が含まれていなければならない。なお、図・表の書き方は物理・化学実験などのレポートの書き方に準じる。

演習01B1
1～30までの奇数の和を表示しよう
1～30までの 奇数 の和　(1+3+5+…+27+29)　を、途中経過を表示しつつ表示せよ。

１．プログラムをエディタで書く
例：

#include <stdio.h>

int main() {
	int num;
	int total=0;

	for(num=1; num<30; num+=2) {
		total += num;
		printf("%dまでの奇数の合計は%d\n", num, total);
	}

	return 0;
}
